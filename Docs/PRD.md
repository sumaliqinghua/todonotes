# 1. 产品定位

Electron 桌面应用，以"仿便签窗口"承载任务树式笔记。任务节点采用块编辑器书写内容，并通过"链接块"进入子任务详情。支持多窗口同时打开同一任务、窗口状态持久化与自动会话恢复。提醒为**应用内弹窗**。

---

# 2. 信息架构（IA）

## 2.1 笔记库面板（Library）

- 管理任务树（类似文件夹树）
- 功能：
  - 新建/重命名/归档/删除（进回收站）
  - 搜索：标题 + 正文块内容（归档可搜；回收站默认不参与搜索，除非进入回收站视图）
  - 打开为便签窗口
  - 可以打钩完成；显示到已完成中

## 2.2 便签窗口（Sticky Window）

- 每个窗口绑定一个 `rootTaskId`
- 窗口内可深入浏览：root → 子任务 → 更深层
- 允许同一任务被多个窗口重复打开（多个窗口实例）

## 2.3 同一路径共享便签 `P0` `已实现`

- 同一个父级及其下所有子任务**共享同一个便签窗口实例**的所有状态（书签、内容等）
- 区别仅在于：从不同子任务右键"打开便签"时，面包屑导航定位到**当前任务对应的位置**
- 即：共享窗口状态，差异化导航入口

# 3. 页面与交互（核心流程）

## 3.1 任务详情页（Task Detail）

包含：

- 顶部自绘标题栏：标题、置顶开关、透明度入口、关闭/最小化
- 导航：返回父级按钮 + 面包屑
- 主体：块编辑器正文（链接块可散落其中；父级不要求是列表）

### 面包屑行为

- 点击祖先节点：**从该节点重新开始浏览路径**（仅重置导航栈，不影响任何数据）

## 3.2 "文本块 → 子任务"转换（明确入口）

P0 至少提供两种触发方式：

- 右键菜单：`转换为子任务`
- 快捷键：`Ctrl/Cmd + Shift + T`（可配置 P1）

转换规则：

1. 以"块"为单位（段落块/列表项块/待办块等）触发转换
2. 创建新 Task：
    - 新 Task 标题 = 原块的文本（可立即编辑）
    - 建立父子关系（见 4.2 edges）
3. 原块替换为"链接块/引用块"（显示子任务标题）
4. 点击链接块 → 进入该子任务详情页（同窗口导航深入）

> 说明：如果未来要支持"段内选中部分文本转换"，作为 P1。

## 3.3 子任务链接块改造（TaskItem 卡片化） `P1`

**现状**：子任务在父任务正文中以简单链接块呈现。

**改造目标**：链接块升级为 **TaskItem 卡片样式**：

- 外观：带包围框（卡片边框），视觉上区别于普通文本块
- 顶部包含 **checkbox**，可直接在父任务正文中勾选完成
- 勾选状态**双向同步到任务树**：
  - 在链接块卡片上打钩 → 任务树中该子任务标记为已完成
  - 在任务树中标记完成 → 链接块卡片上的 checkbox 同步更新
- 卡片内显示子任务标题，点击仍可进入子任务详情页

## 3.4 便签 Checkbox 与子任务双向同步 `P1`

- 便签正文中的 markdown checkbox（如 `- [x] 内容`）与对应子任务的完成状态**双向同步**：
  - 在便签正文中勾选 checkbox → 对应子任务标记为已完成
  - 在任务树/子任务详情中标记完成 → 便签正文中对应 checkbox 自动勾选
- 前提：checkbox 文本需与子任务存在关联（通过链接块或文本匹配）
- 无关联的普通 checkbox 不受影响，仅作为正文内容

## 3.5 链接块 vs 父子关系

- 父子关系**不依赖链接块是否存在**，而是由独立的 `edges`（父子边）存储
- 链接块只是正文里的一个"入口呈现"
- 删除链接块的结果：
  - **不会删除子任务**
  - 只会移除该入口块（该子任务仍在任务树中，仍可从库面板或搜索进入）

### 删除子任务文本后的关联恢复 `P2`

- **现状**：在详情页/便签上删除子任务的链接块文本后，子任务本身仍存在，但正文中的入口丢失
- **方案**：
  - 子任务仍保留在任务树中，可通过库面板或搜索进入
  - P2 提供"重新插入链接块"操作：在正文中右键 → "插入已有子任务链接" → 选择子任务 → 恢复关联入口

## 3.6 子任务弹出为独立便签窗口

- 在子任务页面或链接块菜单提供"在新便签中打开"
- 父窗口内入口仍存在并可点击
- 允许重复打开：同一 Task 可开多个便签窗口

## 3.7 子任务标题重命名 `P1`

- 支持在以下位置直接重命名子任务标题：
  - 任务树面板中（右键 → 重命名 / 双击标题）
  - 父任务正文中的链接块卡片上（双击标题区域编辑）
  - 子任务详情页顶部标题栏
- 重命名后所有引用位置同步更新

## 3.8 拖拽子任务改变层级 `P2`

- 在父任务正文中，支持拖拽链接块（TaskItem 卡片）改变其在正文中的位置
- 在任务树面板中，支持拖拽任务节点改变父子层级关系
- 拖拽改变层级时同步更新 `edges` 表

## 3.9 移动子任务引用 `P2`

- 支持将子任务从一个父任务移动到另一个父任务
- 操作方式：右键链接块 → "移动到..." → 选择目标父任务
- 移动后更新 `edges` 父子关系，原位置链接块移除，目标位置插入新链接块

## 3.10 完成、归档、删除/回收站

- 父任务完成状态**独立**，不随子任务自动变化
- 归档：
  - 从库面板默认隐藏
  - 可搜索、可恢复
- 删除（进回收站）：
  - 任意任务（含已归档）可直接删除进入回收站
  - 删除父任务时，其子任务**一并进入回收站**（递归）
  - 从回收站恢复父任务时，其子任务**一并递归恢复**
  - 进入回收站后：**清除 archived 标记**（避免状态叠加混乱）
  - 回收站保留 30 天，自动清除（物理删除）

### 一键归档已完成子任务 `P2`

- **痛点**：前一天的子任务部分没完成，导致主任务正文越来越长，内容膨胀
- **方案**：在父任务详情页提供"归档已完成子任务"按钮
  - 点击后将所有已完成的子任务批量归档
  - 归档后的子任务从正文链接块中移除（或折叠为"已归档 N 项"摘要）
  - 归档的子任务仍可通过库面板搜索和恢复

# 4. 任务视图与筛选

## 4.1 今日/已过期 Tab `P1`

- 在笔记库面板中新增 **Tab 切换视图**：
  - **今日**：显示截止日期为今天的任务、今日创建的任务
  - **已过期**：显示截止日期已过但未完成的任务
- 每个 Tab 下的任务列表支持点击打开便签窗口
- 过期任务以醒目样式（如红色标记）区分

## 4.2 Pin 任务 `P1`

- 支持将任务"Pin"到笔记库面板顶部
- Pin 的任务始终显示在列表最上方，不受排序规则影响
- Hover 到 Pin 的任务时，**显示完整路径**（面包屑形式的 tooltip）
- 操作方式：右键 → "Pin 到顶部" / "取消 Pin"

## 4.3 等待中/阻塞状态与自定义标签 `P2`

- 支持为任务添加**自定义标签**（如"等待中"、"阻塞"、"进行中"等）
- 标签可用于筛选和分组显示
- 预设常用标签：等待中、阻塞、进行中
- 支持用户自定义新标签（名称 + 颜色）
- 可按标签/依赖关系聚合显示任务

## 4.4 文本快捷标记（Emoji 标记） `P2`

- 在任务标题或正文前添加**快捷 emoji 标记**，用于视觉分类
- 提供**预设标记**（如 🔴 紧急、🟡 进行中、🟢 已完成、🚫 阻塞、📌 关注等）
- 支持**自定义标记**：用户可添加自定义 emoji + 提示文字
- Hover 到标记上显示 **tooltip 提示**（如 hover 🔴 显示"紧急"）
- 标记仅作为视觉辅助，不附带额外逻辑功能

## 4.5 一键隐藏已勾选内容 `待定`

- 在便签正文中提供开关，隐藏所有已勾选的 checkbox 项
- 减少视觉干扰，聚焦未完成内容
- 隐藏后可通过开关恢复显示

> 此需求待进一步评估必要性和交互方式。

# 5. 数据与一致性规则（技术关键点）

## 5.1 会话恢复：浏览路径要存"栈"

每个窗口实例保存：

- `windowId`（实例 ID）
- 位置/大小/透明度/置顶
- `rootTaskId`
- `navPathTaskIds: string[]`
  例如 `[A, B, C]` 表示当前浏览在 C，面包屑可还原 A/B/C

保存时机：

- 仅在应用退出时持久化所有窗口状态（P0）

启动时：

- 自动恢复上次所有窗口实例集合（包括同一任务的重复窗口）

## 5.2 父子关系：edges 表独立

- `edges(parentTaskId, childTaskId, createdAt, ...)`
- `blocks` 中的链接块可引用 `childTaskId`，但不是唯一入口，也不是关系来源
- 任务正文块存储在 `tasks.blocks` JSON（不做 blocks 表）
- FTS 索引由应用层在写入时生成/更新（非数据库触发器）

## 5.3 多窗口编辑一致性

P0 策略：

- **最后写入胜出（Last-write-wins）**
- 需要做两件事避免"看见过时内容"：
  1. 数据层写入后广播 `taskUpdated(taskId)` 事件
  2. 所有打开该 task 的窗口收到后自动刷新并覆盖本地视图（无提示）

冲突提示（P1）：可加入"检测到并发编辑"提醒与合并策略，但 P0 不做。

## 5.4 提醒调度与过期提醒

- 应用运行中：到点弹应用内弹窗
- 应用未运行：下次启动时执行"过期检查"
  - 扫描所有未确认提醒，若 `reminderAt <= now`，弹出"过期提醒"
  - 弹窗策略：P0 **合并为一个弹窗列表**，点击条目打开对应任务

## 5.5 附件命名策略

- 实际文件名：UUID（防同名覆盖）
- 显示名：保留原始文件名（存在块数据里）
- 存放目录：应用数据目录下按日期/类型分桶

# 6. 便签窗口 UX 细化

## 6.1 吸附行为定义（P0）

- 拖拽接近屏幕边缘 **20px** 时自动贴边
- 窗口不允许完全移出可视区域（至少保留标题栏可拖回）
- 吸附是"拖拽过程即时贴边"，松手后保持该位置

## 6.2 透明度范围与默认值（P0）

- 默认：100%
- 可调范围：30% ~ 100%
- 低于 30% 不允许（直接限制下限）

# 7. AI 功能 `P3`

## 7.1 AI 今日总结

- 对今日新增的文本内容进行 AI 摘要总结
- 入口：笔记库面板或便签窗口中提供"今日总结"按钮
- 总结范围：当天创建或修改的所有任务内容
- 输出：生成结构化摘要，可保存为新任务或复制到剪贴板

## 7.2 AI 快速提问

- 在任务编辑过程中，支持快速向 AI 提问
- 可配置搜索引擎辅助（快速搜索浏览器）
- 入口：快捷键或工具栏按钮唤起提问面板
- AI 回答可直接插入到当前正文中或作为浮窗展示

## 7.3 AI 拆子任务 `远期`

- 选中一段文本，AI 自动拆分为多个子任务并创建
- 暂不纳入近期迭代，作为远期规划

# 8. 远期规划（Backlog）

以下功能暂不纳入近期迭代，记录备忘：

| 功能 | 说明 |
|------|------|
| 优先级/预计时间/番茄时钟 | 任务增加优先级字段、预估时间、内置番茄钟计时 |
| 定位上次光标位置 | 重新打开文档时自动定位到上次编辑的光标位置 |
| 本地可读文档导出 | 将任务数据导出为 markdown 等可读格式，方便外部 AI 工具读取 |
