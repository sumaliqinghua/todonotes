  # 1. 产品定位

  Electron 桌面应用，以“仿便签窗口”承载任务树式笔记。任务节点采用块编辑器书写内容，并通过“链接块”进入子任务详情。支持多窗口同时打开同一任务、窗口状态持久化与自动会话恢复。提醒为**应用内弹窗**。

  ---

  # 2. 信息架构（IA）

  ## 2.1 笔记库面板（Library）

  - 管理任务树（类似文件夹树）
  - 功能：

    - 新建/重命名/归档/删除（进回收站）
    - 搜索：标题 + 正文块内容（归档可搜；回收站默认不参与搜索，除非进入回收站视图）
    - 打开为便签窗口
    - 可以打钩完成；显示到已完成中

  ## 2.2 便签窗口（Sticky Window）

  - 每个窗口绑定一个 `rootTaskId`
  - 窗口内可深入浏览：root → 子任务 → 更深层
  - 允许同一任务被多个窗口重复打开（多个窗口实例）

  ---

  # 3. 页面与交互（核心流程）

  ## 3.1 任务详情页（Task Detail）

  包含：

  - 顶部自绘标题栏：标题、置顶开关、透明度入口、关闭/最小化
  - 导航：返回父级按钮 + 面包屑
  - 主体：块编辑器正文（链接块可散落其中；父级不要求是列表）

  ### 面包屑行为

  - 点击祖先节点：**从该节点重新开始浏览路径**（仅重置导航栈，不影响任何数据）

  ## 3.2 “文本块 → 子任务”转换（明确入口）

  P0 至少提供两种触发方式（按你的建议）：

  - 右键菜单：`转换为子任务`
  - 快捷键：`Ctrl/Cmd + Shift + T`（可配置 P1）

  转换规则：

  1. 以“块”为单位（段落块/列表项块/待办块等）触发转换
  2. 创建新 Task：

      - 新 Task 标题 \= 原块的文本（可立即编辑）
      - 建立父子关系（见 4.2 edges）
  3. 原块替换为“链接块/引用块”（显示子任务标题）
  4. 点击链接块 → 进入该子任务详情页（同窗口导航深入）

  > 说明：如果未来要支持“段内选中部分文本转换”，作为 P1。
  >

  ## 3.3 链接块 vs 父子关系

  - 父子关系**不依赖链接块是否存在**，而是由独立的 `edges`（父子边）存储。
  - 链接块只是正文里的一个“入口呈现”。
  - 删除链接块的结果：

    - **不会删除子任务**
    - 只会移除该入口块（该子任务仍在任务树中，仍可从库面板或搜索进入）

  ## 3.4 子任务弹出为独立便签窗口

  - 在子任务页面或链接块菜单提供“在新便签中打开”
  - 父窗口内入口仍存在并可点击
  - 允许重复打开：同一 Task 可开多个便签窗口

  ## 3.5 完成、归档、删除/回收站（补充边界）

  - 父任务完成状态**独立**，不随子任务自动变化
  - 归档：

    - 从库面板默认隐藏
    - 可搜索、可恢复
  - 删除（进回收站）：

    - 任意任务（含已归档）可直接删除进入回收站
    - 进入回收站后：建议**清除 archived 标记**（避免状态叠加混乱）
    - 回收站保留 30 天，自动清除（物理删除）

  # 4. 数据与一致性规则（技术关键点）

  ## 4.1 会话恢复：浏览路径要存“栈”

  每个窗口实例保存：

  - `windowId`（实例 ID）
  - 位置/大小/透明度/置顶
  - `rootTaskId`
  - `navPathTaskIds: string[]`
    例如 `[A, B, C]` 表示当前浏览在 C，面包屑可还原 A/B/C

  启动时：

  - 自动恢复上次所有窗口实例集合（包括同一任务的重复窗口）

  ## 4.2 父子关系：edges 表独立

  - `edges(parentTaskId, childTaskId, createdAt, ...)`
  - `blocks` 中的链接块可引用 `childTaskId`，但不是唯一入口，也不是关系来源

  ## 4.3 多窗口编辑一致性（补充策略）

  P0 策略：

  - **最后写入胜出（Last-write-wins）**
  - 需要做两件事避免“看见过时内容”：

    1. 数据层写入后广播 `taskUpdated(taskId)` 事件
    2. 所有打开该 task 的窗口收到后自动刷新（或提示“已更新，点击刷新”——P1）

  冲突提示（P1）：可加入“检测到并发编辑”提醒与合并策略，但 P0 不做。

  ## 4.4 提醒调度与过期提醒（补边缘场景）

  - 应用运行中：到点弹应用内弹窗
  - 应用未运行：下次启动时执行“过期检查”

    - 扫描所有未确认提醒，若 `reminderAt <= now`，弹出“过期提醒”
    - 弹窗策略：

      - P0：逐个弹或合并为列表弹窗（二选一；建议合并，避免连环弹）
      - 你若不指定，我建议：**合并为一个弹窗列表**，点击条目打开对应任务

  ## 4.5 附件命名策略

  - 实际文件名：UUID（防同名覆盖）
  - 显示名：保留原始文件名（存在块数据里）
  - 存放目录：应用数据目录下按日期/类型分桶（实现细节可后定）

  # 5. 便签窗口 UX 细化

  ## 5.1 吸附行为定义（P0）

  - 拖拽接近屏幕边缘 **20px** 时自动贴边
  - 窗口不允许完全移出可视区域（至少保留标题栏可拖回）
  - 吸附是“拖拽过程即时贴边”，松手后保持该位置

  ## 5.2 透明度范围与默认值（P0）

  - 默认：100%
  - 可调范围：30% \~ 100%
  - 低于 30%不允许（或给出提示并自动回弹到 30%——二选一；建议直接限制下限）

  # 6. P0 / P1 里程碑

  ## P0（可用闭环）

  - 任务树 + 搜索（标题+正文）
  - 便签窗口（无边框、拖拽/缩放、置顶、透明度、吸附）
  - 块编辑器基础块 + Markdown 粘贴解析（基础）
  - 块→子任务转换（右键 + 快捷键）
  - 链接块导航 + 返回父级 + 面包屑
  - edges 独立存父子关系（删除链接块不删子任务）
  - 归档 / 回收站（30 天清除）
  - 提醒（应用内弹窗 + 启动时过期检查）
  - 会话恢复（按窗口实例 + 恢复 navPathTaskIds）

  ## P1（增强）

  - Snooze（5/10/30）
  - 并发编辑冲突提示/更细刷新策略
  - 更完整块类型（表格等，是否提升优先级取决于画像）
  - 任务树拖拽改层级、批量操作
  - 搜索摘要高亮与更丰富过滤
