# 功能补充建议清单（基于当前实现评估）

更新时间：2026-02-07  
评估范围：Electron 桌面端当前代码基线 + 现有界面（任务树、面包屑、书签便签、块编辑器）

---

## 1. 现状简评

当前版本已经具备了很好的“任务拆分 + 子任务跳转 + 多窗口便签”基础，尤其是：

- 任务树 + 面包屑 + 子任务链接块已经形成主导航闭环。
- 便签窗口支持置顶、皮肤、书签、番茄钟，适合“边做边记”。
- 数据层模型（tasks/edges/reminders/attachments/fts/window_states）已经为后续功能扩展预留了空间。

目前的主要缺口不在“能不能记”，而在“能不能持续高效推进任务”：

- 计划层（优先级、截止时间、依赖、批量操作）较弱。
- 执行层（提醒入口、附件入口、搜索高亮、粘贴解析）仍有断点。
- 复盘层（历史、统计、导出、版本）能力较少。

---

## 2. 优先级建议（先补闭环，再做增强）

### P0（建议优先落地，直接影响日常可用性）

1. **提醒设置 UI 闭环**
   - 增加“创建/查看/删除提醒”入口（任务详情与便签都可操作）。
   - 与现有 `reminder:create/reminder:listByTask/reminder:delete` 无缝对接。

2. **附件 UI 闭环**
   - 增加上传按钮、附件列表、在文件夹中显示。
   - 支持拖拽上传（后续可叠加），先完成基础入口。

3. **Markdown 粘贴自动解析**
   - 当前已有复制为 Markdown，建议补齐“粘贴即结构化”。
   - 优先支持标题、列表、任务列表、引用、代码块。

4. **搜索结果高亮摘要**
   - 搜索结果显示命中片段，不只显示标题。
   - 可减少“点开才知道是不是想找的内容”的来回切换。

5. **任务树显示子任务数量与进度**
   - 每个任务显示“子任务总数/完成数”，提升管理可见性。

6. **链接块显示状态信息**
   - `taskLink` 显示“完成/未完成”视觉状态（颜色或图标）。
   - 减少打开子任务确认状态的频次。

7. **操作反馈与撤销保护**
   - 删除/归档/恢复提供 toast + 撤销窗口（如 5 秒）。
   - 避免误操作造成心智负担。

8. **快捷键帮助面板**
   - 提供 `?` 或 `Cmd/Ctrl + /` 调出快捷键说明。

9. **任务树展开状态持久化**
   - 现在切换后折叠状态容易丢失；建议按窗口保存。

10. **回收站批量操作**
   - 批量恢复、批量永久删除、清空回收站。

---

### P1（建议第二阶段，聚焦效率提升）

11. **任务树拖拽改层级/排序**
   - 支持拖拽变更父子关系与同级排序。

12. **同级手动排序字段**
   - 增加 `sort_order`，避免完全依赖更新时间排序。

13. **批量多选操作**
   - 多选后统一完成、归档、删除、移动。

14. **任务元数据：优先级/截止时间/预计时长**
   - 让任务从“笔记对象”升级为“可调度对象”。

15. **标签系统（轻量）**
   - 支持标签筛选与组合过滤（如 `#工作 #紧急`）。

16. **智能视图（Today/Next/Waiting）**
   - 从树结构切换到“工作台视图”，降低全局扫描成本。

17. **收集箱清理流程（Inbox Triage）**
   - 一键流程：分类、拆分、设提醒、归档无效条目。

18. **提醒 Snooze（5/10/30 分钟）**
   - 与提醒弹窗联动，减少“现在没空处理”的打断成本。

19. **重复提醒/周期任务**
   - 支持每天/每周规则，避免重复手工创建。

20. **任务模板**
   - 如“需求评审模板”“Bug 排查模板”“发布模板”。

21. **命令面板（Cmd/Ctrl + K）**
   - 统一入口：创建任务、跳转任务、打开窗口、归档等。

22. **快速跳转（全局任务搜索）**
   - 不进入搜索页直接跳转目标任务。

23. **书签栏增强**
   - 支持拖拽排序、固定高频项、分组（项目/阶段）。

24. **面包屑“兄弟任务预览”增强**
   - 弹层显示兄弟任务状态（完成、提醒、更新时间）。

25. **便签窗口“仅阅读模式/专注模式”**
   - 减少误触编辑，适合会议投屏或复盘场景。

26. **番茄钟与任务绑定记录**
   - 每次专注记录归档到任务（累计专注时长）。

---

### P2（建议第三阶段，面向长期能力）

27. **并发编辑冲突提示（非覆盖式）**
   - 在多窗口编辑同任务时提示差异块，支持选择合并或覆盖。

28. **历史版本快照**
   - 按时间点查看任务内容变更，可回滚。

29. **自动备份与恢复向导**
   - 定时备份数据库 + 附件目录，支持一键恢复。

30. **导入导出（Markdown/JSON）**
   - 支持单任务与整库导出，便于迁移与归档。

31. **任务引用关系图（反向链接）**
   - 查看“哪些任务链接到了我”，形成知识网络。

32. **依赖关系（阻塞/被阻塞）**
   - 区分“子任务”与“依赖任务”，用于复杂计划。

33. **工作流规则自动化**
   - 例如：任务完成后自动归档子项、提醒到期自动升优先级。

34. **附件增强（预览/OCR/快速打开）**
   - 至少支持图片与文本预览，提升资料检索效率。

35. **统计面板（周报级）**
   - 新增/完成趋势、专注时长、活跃项目、滞留任务。

36. **跨设备同步（可选）**
   - 后续可先做本地文件同步，再评估云同步。

37. **插件扩展点（长期）**
   - 为自定义块、自动化脚本、外部系统接入留接口。

38. **权限与隐私增强**
   - 本地加密、锁屏密码、敏感任务隐藏。

39. **无障碍增强**
   - 完整键盘导航、ARIA、字号缩放。

40. **多语言与主题体系**
   - 提供中文/英文切换与深浅主题可配置。

## 3. 可立即进入开发排期的组合包

### 组合包 A（2~3 周，立刻提效）

- 提醒 UI 闭环
- 附件 UI 闭环
- Markdown 粘贴解析
- 搜索高亮摘要
- 删除撤销 toast

**产出价值**：把当前系统最明显的“断点”补齐，减少跳出编辑流。

### 组合包 B（3~4 周，强化任务管理）

- 拖拽改层级 + 手动排序
- 多选批量操作
- 优先级/截止时间/标签
- 智能视图（Today/Next）

**产出价值**：把“能记录”升级为“能计划和推进”。

### 组合包 C（4+ 周，长期壁垒）

- 冲突提示 + 历史快照
- 导入导出 + 自动备份
- 依赖关系 + 统计面板

**产出价值**：提升可靠性与长期可持续使用能力。

## 4. 与现有代码结构的衔接建议

- **数据层（SQLite）**
  - 可新增字段：`priority`、`due_at`、`sort_order`、`estimated_minutes`。
  - 可新增表：`task_tags`、`task_tag_relations`、`task_dependencies`、`task_snapshots`。

- **IPC 层**
  - 当前 typed IPC 结构已清晰，建议按功能域继续扩展 channel（如 `task:reorder`、`task:batchUpdate`、`task:saveSnapshot`）。

- **渲染层**
  - `LibraryPanel`：拖拽、多选、进度徽标、智能视图入口。
  - `TaskDetail/StickyView`：提醒与附件入口、命令面板、链接状态徽标。
  - `ReminderModal`：增加 Snooze 与“标记完成”。

- **稳定性**
  - 保持“小步可验收”策略，先闭环前端入口，再做高级交互。
  - 对并发编辑与历史版本优先补测试用例。

---

## 5. 建议优先级结论（可直接用于下一轮计划）

如果只选 5 项优先落地，我建议是：

1. 提醒 UI 闭环
2. 附件 UI 闭环
3. Markdown 粘贴解析
4. 搜索高亮摘要
5. 任务树拖拽改层级 + 排序

这 5 项能最大化提升“从记录到执行”的连续性，也是当前产品从“好用原型”走向“可长期依赖工具”的关键一步。
