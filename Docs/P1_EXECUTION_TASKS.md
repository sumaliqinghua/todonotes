更新时间：2026-02-07

## 1. 目标与范围

本清单将 P1 建议从“想法列表”转为“可开发任务”，用于直接进入迭代排期。

- 覆盖范围：任务树效率、任务调度、提醒增强、模板与快速入口、便签导航增强。
- 不含范围：云同步、多人协作、插件系统、跨端架构调整（保留到 P2+）。
- 执行原则：每个里程碑 2~5 分钟可验收，优先小步可交付。

---

## 2. 里程碑执行任务（建议排期）

## P1-M1：任务树结构与批量效率（建议 1~1.5 周）

覆盖建议：11、12、13

### 任务清单
- [ ] 数据层：新增 `sort_order` 字段并补迁移脚本，保持旧数据兼容。
- [ ] IPC：新增 `task:reorder`、`task:batchUpdate` 接口。
- [ ] 任务树：支持拖拽改层级（父子变更）与同级排序。
- [ ] 任务树：支持多选（Shift/Cmd）与批量完成、归档、删除。
- [ ] 交互反馈：拖拽落点高亮、批量操作确认与结果提示。
- [ ] 测试：覆盖排序持久化、跨层级移动、批量操作原子性。

### 验收标准
- [ ] 拖拽后刷新/重启仍保持顺序与层级。
- [ ] 多选批量操作后，树与详情状态一致。
- [ ] 误操作可撤销或有明确确认。

## P1-M2：任务元数据与调度视图（建议 1.5~2 周）

覆盖建议：14、15、16、17

### 任务清单
- [ ] 数据层：`tasks` 增加 `priority`、`due_at`、`estimated_minutes`。
- [ ] 数据层：新增标签表与任务标签关系表。
- [ ] IPC：新增 `task:updateMeta`、`tag:list`、`tag:bind`、`tag:unbind`。
- [ ] 详情页：增加“任务属性面板”（优先级/截止时间/预计时长/标签）。
- [ ] 视图层：新增智能视图（Today/Next/Waiting）。
- [ ] 收集箱：新增 Triage 流程（分类、拆分、设提醒、归档）。
- [ ] 测试：截止日期筛选、标签组合过滤、属性更新一致性。

### 验收标准
- [ ] 属性修改实时生效且重启后保留。
- [ ] 智能视图结果符合规则且可快速跳转任务。
- [ ] 收集箱可以在一个流程内完成“从输入到归类”。

## P1-M3：提醒增强、模板与快速入口（建议 1.5~2 周）

覆盖建议：18、19、20、21、22

### 任务清单
- [ ] 提醒弹窗：新增 Snooze 5/10/30 分钟。
- [ ] 提醒系统：新增周期提醒（每天/每周）。
- [ ] 模板：新增模板管理（创建、应用、删除）。
- [ ] 快捷入口：新增命令面板（Cmd/Ctrl + K）。
- [ ] 快速跳转：支持全局任务搜索直达。
- [ ] IPC：新增 `reminder:snooze`、`reminder:createRecurring`、`template:*`。
- [ ] 测试：Snooze 二次触发、周期规则正确性、命令面板动作可达性。

### 验收标准
- [ ] 提醒可一键延后并按新时间再次触发。
- [ ] 周期提醒在多个周期稳定生效。
- [ ] 命令面板可在 3 次键盘操作内完成常用动作。

## P1-M4：便签导航与专注体验增强（建议 1~1.5 周）

覆盖建议：23、24、25、26

### 任务清单
- [ ] 书签栏：支持拖拽排序、固定置顶、分组。
- [ ] 面包屑兄弟任务弹层：显示状态（完成/提醒/最近更新时间）。
- [ ] 便签模式：新增只读模式与专注模式（弱化控制栏）。
- [ ] 番茄钟：记录专注会话到任务日志（次数、总时长）。
- [ ] 测试：书签排序持久化、模式切换无数据丢失、番茄记录准确。

### 验收标准
- [ ] 高频书签可稳定固定且重启后不乱序。
- [ ] 只读模式下不可误编辑，退出后恢复编辑能力。
- [ ] 番茄会话数据可在任务中查看并累计。

## 4. 实施依赖与风险提醒

- **数据迁移风险**：`tasks` 增字段与排序逻辑调整，需提供迁移回滚方案。
- **交互复杂度风险**：拖拽 + 多选并存时容易产生冲突，建议分阶段开关上线。
- **性能风险**：智能视图、全局命令搜索可能增加查询压力，建议优先复用 FTS 并加索引。
- **一致性风险**：多窗口场景下批量操作与排序要保证广播更新正确。

---

## 5. 推荐执行顺序（最小风险）

1. 先做 `P1-M1`（树结构与批量能力），夯实基础操作。
2. 再做 `P1-M2`（元数据与视图），建立调度能力。
3. 然后做 `P1-M3`（提醒/模板/命令面板），提升执行效率。
4. 最后做 `P1-M4`（便签体验增强），打磨高频使用体验。

该顺序可以最大化“每一阶段都有可见收益”，同时减少返工。
