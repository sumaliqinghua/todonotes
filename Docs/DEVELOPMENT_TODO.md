## 默认方案记录（实现中已采用）

- 状态管理暂用 React 本地状态，未引入 Zustand；后续如需跨窗口共享再评估。
- 样式暂用 CSS Variables + 手写样式，未引入 Tailwind。
- 已补充 `.gitignore`，但未初始化 Git 仓库。
- 任务树支持缩进与折叠，暂未实现拖拽改层级。
- 父子关系按“单父节点”处理（祖先链取最早建立的 parent）。
- 块→子任务转换当前仅支持段落/标题块；列表项/待办块待补。
- Markdown 粘贴解析未实现，后续可加粘贴拦截解析。
- 提醒 UI 当前暂时隐藏，仅保留到期弹窗能力。
- 附件 UI 已移除（当前版本不做）。
- 当前主窗口不显示透明度/置顶按钮；置顶仅用于便签窗口。
- 仅保留一个主窗口实例；可打开多个置顶便签。
- 子任务展示迁移到左侧任务树缩进列表，详情页不再显示子任务区/返回父级。
- 附件与提醒面板从详情页移除，详情页/便签以文档编辑为主。
- 视图切换（已完成/归档/回收站）暂不提供。
- 便签仅保留面包屑导航（不显示返回父级按钮）。

## 开发顺序建议

1. **阶段 0** → 项目初始化，搭建骨架
2. **阶段 1 + 7** → 数据层 + IPC，确保数据流通
3. **阶段 2** → 窗口系统，确保多窗口可用
4. **阶段 4 + 5** → 编辑器 + 详情页，核心编辑体验
5. **阶段 3** → 库面板，任务管理入口
6. **阶段 6** → 提醒系统
7. **阶段 8 + 9** → UI完善 + 跨平台
9. **P1** → 迭代增强
