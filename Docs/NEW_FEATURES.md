# 1. 子任务交互增强

## 1.1 链接块卡片化（TaskItem 样式） `P1`

**方案**：链接块升级为 TaskItem 卡片样式：

- 外观：带上当前的包围框，视觉上区别于普通文本块；勾选checkbox后同样有划线效果
- 左侧包含 **checkbox**，可直接在父任务正文中勾选完成
- 勾选状态**双向同步到任务树**：
  - 卡片上打钩 → 任务树中该子任务标记为已完成
  - 任务树中标记完成 → 卡片 checkbox 同步更新
- 卡片内显示子任务标题，点击进入子任务详情页

## 1.2 便签 Checkbox 与子任务双向同步 `P1`

- 便签正文中的 markdown checkbox（如 `- [x] 内容`）与对应子任务完成状态**双向同步**：
  - 正文中勾选 checkbox → 对应子任务标记已完成
  - 任务树/详情中标记完成 → 正文 checkbox 自动勾选
- 前提：checkbox 需与子任务存在关联（通过链接块或文本匹配）
- 无关联的普通 checkbox 不受影响

## 1.3 子任务标题重命名 `P1`

- 支持在以下位置直接重命名：
  - 任务树面板（右键 → 重命名 / 双击标题）
  - 子任务详情页顶部标题栏
- 重命名后所有引用位置同步更新

## 1.4 删除链接块后的关联恢复 `P2`

**痛点**：删除子任务链接块文本后，子任务仍存在但正文入口丢失。

**方案**：
- 子任务保留在任务树中，可通过库面板或搜索进入
- 提供"重新插入链接块"操作：正文右键 → "插入已有子任务链接" → 选择子任务 → 恢复入口

## 1.5 拖拽子任务改变层级 `P2`

- 任务树面板中：拖拽任务节点改变父子层级关系
- 拖拽时同步更新 `edges` 表

## 1.6 移动子任务引用 `P2`

- 将子任务从一个父任务移动到另一个父任务
- 操作：右键链接块 → "移动到..." → 选择目标父任务
- 移动后更新 `edges` 父子关系，原位置链接块移除，目标位置插入新链接块

## 1.7 一键归档已完成子任务 `P2`

**痛点**：前一天的子任务部分没完成，导致主任务正文越来越长，内容膨胀。

**方案**：在父任务详情页提供"归档已完成子任务"按钮：
- 点击后将所有已完成的子任务批量归档
- 归档后的子任务从正文链接块中移除
- 归档的子任务仍可通过库面板搜索和恢复

# 3. AI 功能

## 3.1 AI 今日总结 `P3`

- 对今日新增/修改的文本内容进行 AI 摘要总结
- 入口：笔记库面板或便签窗口中"今日总结"按钮
- 总结范围：当天创建或修改的所有任务内容
- 输出：结构化摘要，可保存为新任务或复制到剪贴板

## 3.2 AI 快速提问 `P3`

- 任务编辑中快速向 AI 提问
- 可配置搜索引擎辅助（快速搜索浏览器）
- 入口：快捷键或工具栏按钮唤起提问面板
- AI 回答可插入正文或浮窗展示

## 3.3 AI 拆子任务 `远期`

- 选中一段文本，AI 自动拆分为多个子任务并创建
- 暂不纳入近期迭代
