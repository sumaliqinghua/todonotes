## [M0.8] Session Release 改为精简发布流（AI摘要/提交/飞书）

- **改了什么**：
  - 重写 `session_release.py`，默认流程改为：自动加载 `.env` → 生成 `ai-summary.md` → Git 提交 → 飞书通知。
  - 移除 dev 启动、截图、diff 细节报告等非核心步骤与产物；默认输出目录改为 `.git/session-release`，避免污染业务目录 `checkpoints`。
  - 同步更新 skill 文档：`SKILL.md`、`references/usage.md`、`references/feishu.md`。
- **为什么这么改**：
  - 用户明确要求只保留“AI总结 + 提交 + 飞书”链路，减少无意义中间产物与噪声。
  - 自动读取 `.env` 可以显著降低执行门槛，避免重复手动注入环境变量。
- **已知限制**：
  - 当前“AI总结”为规则化离线摘要，不是外部大模型生成。
  - 若改动规模极大，飞书文本会截断摘要（保留前 1200 字符）。
- **关联假设**：
  - `[2026-02-07/M0.8] AI 摘要采用规则化离线生成`
