  - 验收标准：链接块视觉为“内联卡片 + checkbox”，在两个窗口中表现一致

  - 具体执行步骤：
    1. 在编辑器内勾选链接块时触发 `task:update(isCompleted)`
    2. 监听任务变更时回写并刷新正文中对应链接块完成态
    3. 保持现有任务树 checkbox 行为不回退
  - 验收标准：任一入口切换完成状态后，任务树与正文链接块均保持同步
  - 依赖关系：依赖“链接块卡片化”任务

- [x] 执行内容：实现 markdown checkbox 与子任务完成状态双向同步（仅关联项生效）
  - 具体执行步骤：
    1. 建立“markdown checkbox ↔ 子任务”关联映射（链接块优先 + 文本匹配）
    2. 在正文勾选关联 checkbox 时同步更新子任务完成态
    3. 在任务完成态变化时回写关联 checkbox 文本
  - 验收标准：关联项双向同步；无关联普通 checkbox 保持原样
  - 依赖关系：依赖“链接块卡片化”与“双向同步”任务

- [x] 执行内容：实现子任务标题重命名增强（`Library` 顶栏 + `Sticky` 顶栏）及引用同步
  - 具体执行步骤：
    1. 支持在 `TaskDetail` 顶部标题栏直接编辑
    2. 支持在 `Sticky` 顶部标题栏直接编辑
    3. 重命名后同步更新 `taskLink` 标题、已关联 markdown checkbox 文本、sticky 书签标题
  - 验收标准：两个入口均可重命名，所有引用位置实时更新

- [x] 执行内容：新增子任务重名约束与用户提示（不支持同名子任务）
  - 具体执行步骤：
    1. 在子任务创建与重命名链路增加重名校验
    2. 对冲突场景提供前端提示并阻止写入
    3. 保证不会产生新的重名子任务
  - 验收标准：重复名称提交失败且有可理解提示
