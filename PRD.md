## 1. 产品定位

Electron 桌面任务笔记应用，采用“库面板（Library）+ 置顶便签（Sticky）”双窗口形态：
- `Library` 负责任务树管理、搜索与任务状态管理。
- `Sticky` 负责沉浸式编辑、快速导航和置顶便签工作流。
- 任务正文使用块编辑器（Tiptap），通过链接块承载“子任务入口”。

---

## 2. 当前代码基线（截至 2026-02-07）

### 2.1 技术栈
- 主进程：Electron + TypeScript
- 渲染进程：React 18 + Zustand
- 编辑器：Tiptap（StarterKit + TaskList + TaskItem + 自定义 TaskLink）
- 数据层：better-sqlite3 + FTS5

### 2.2 数据模型
- `tasks`：任务主表（含 `blocks` JSON、完成/归档/回收状态）
- `edges`：父子关系表（关系独立于正文链接块）
- `window_states`：窗口状态与导航路径
- `sticky_bookmarks`：便签书签栏
- `reminders`：提醒
- `attachments`：附件元数据
- `tasks_fts`：全文检索索引（应用层写入维护）

## 4. 核心业务规则（当前有效）

- 父子关系以 `edges` 为准，链接块仅为入口展示。
- 删除链接块不会影响 `edges` 与子任务实体。
- 删除父任务会递归进入回收站；恢复父任务会递归恢复子任务。
- 进入回收站时清除归档标记，避免状态叠加混乱。
- 多窗口编辑一致性：写入后广播，其他窗口自动刷新覆盖。
- sticky 共享根规则：同一顶层父任务下多个便签共享书签与皮肤状态。
- sticky 打开规则：新开便签定位到目标任务路径，但共享根保持顶层父任务。
- 会话恢复按窗口实例恢复，包含 `rootTaskId + navPathTaskIds`。

### 5.3 带时间的待处理工作台（M0.13-R21 已实现首轮）
1. 目标与统一入口
   - 本轮目标不是再新增一套“时间面板”，而是把现有 Sticky 的“待处理”升级为“带时间的块级工作台”。
   - Sticky 继续作为“执行工作台”：
     - 手动待处理：表示“我先把这段内容挂起，稍后回来处理”。
     - 截止时间：表示“这段内容最晚应在这个时间点前完成”。
   - 两者共用 Sticky 的同一个入口，避免把待处理、时间、提醒拆成三套并行入口。
2. 数据范围与字段定义
   - 时间挂在“正文块”上，不挂在整个任务上。
   - 这里的“正文块”指编辑器中的单条内容块，首轮包含：
     - `paragraph`：普通段落
     - `heading`：标题
     - `listItem`：普通列表项
     - `taskItem`：checkbox 任务项
   - 首轮不支持对子任务链接卡片 `taskLink` 直接设置时间。
   - 每个正文块最多只有一个时间字段：
     - `dueAt`：截止时间，表示“最晚在这个时间点前完成它”
3. Sticky 右键入口
   - 在正文块右键菜单中补充时间操作：
     - `添加文本块到待处理`：保留现有纯手动待处理能力
     - `设置截止时间...`：二级菜单，包含：
       - `5分钟后截止`
       - `10分钟后截止`
       - `30分钟后截止`
       - `1小时后截止`
       - `2小时后截止`
       - `自定义截止时间...`
     - `清除截止时间`：移除当前块的 `dueAt`
   - `自定义截止时间...` 窗口只需要一个字段：
     - `日期时间`：精确到分钟的本地时间输入
   - 从右键菜单进入自定义时间时，默认时间值使用“当前时间”，不预填未来 1 小时。
   - 如果同一个块已经在队列中，再次设置时间时应更新原条目，而不是新增重复条目。
4. 正文块前的显示规则
   - 带时间的正文块在正文前显示一个时间徽标和倒计时。
   - 颜色区分：
     - `截止时间`：使用暖色系（例如红色或橙色）表示“离截止还剩多久 / 已超时”
   - 倒计时格式统一为 `xxdxxhxxm`，其中值为 `0` 的位不显示：
     - 例如 `2d3h5m`
     - 例如 `4h20m`
     - 例如 `8m`
   - 截止时间未到时直接显示剩余时间，例如 `10m`
   - 截止时间到点或超过后显示 `已超时` 或 `超时xxm`
5. Sticky 待处理弹层与筛选
   - Sticky 书签栏继续只保留一个 `待处理 (n)` 入口。
   - `全部` 视图同时显示：
     - 有时间的块：按截止时间从近到远排序
     - 纯手动待处理块：排在有时间块之后，并保留当前的手动顺序/拖拽顺序
   - `今天` 视图只显示有时间的块，且其 `dueAt` 落在用户本地“今天”范围内。
   - `1 小时内` 视图只显示有时间的块，且其 `dueAt` 距离当前时间不超过 60 分钟。
   - 纯手动待处理块没有时间属性，因此只出现在 `全部` 视图，不出现在 `今天` 和 `1 小时内`。
6. 快速跳转与最近顺序
   - Sticky 底部左右切换按钮继续保留，但顺序改为基于当前筛选结果。
   - 在 `今天` 或 `1 小时内` 视图下，左右切换按时间最近顺序轮换：
     - 已超时但尚未清除的块优先
     - 未来块按离现在最近的截止时间排在前面
   - 在 `全部` 视图下：
     - 先按时间顺序切换有时间的块
     - 再进入纯手动待处理块
7. 刷新与倒计时更新
   - Sticky 当前页面内容在以下时机刷新：
     - 切换到新页面时立即刷新
     - 每 2 分钟自动刷新当前页面内容一次
   - 文本前倒计时文案在前端本地按 1 分钟粒度更新，并在页面内容刷新后重新校正。
   - 这样可以同时保证：
     - 多窗口场景下页面内容不会长时间陈旧
     - 倒计时不会因为 2 分钟才刷新一次而明显跳字
8. 本轮明确不做的范围
   - 不把该能力接到 `reminders` 表或提醒弹窗闭环里。
   - 不在 Library 新增 Today/1 小时内视图。
   - 不新增任务级截止时间。
   - 不支持对子任务链接卡片 `taskLink` 直接挂时间。

### 5.4 子任务交互增强（`Docs/NEW_FEATURES.md` #1.1~1.3）
1. 链接块保持“内联形态”，增强为带 checkbox 的卡片样式，并在 `Library` 详情页与 `Sticky` 中一致支持。
2. 链接块 checkbox 与任务完成状态双向同步：卡片勾选可更新任务树，任务树/详情页勾选可回写卡片展示。
3. 便签正文 markdown checkbox 与子任务完成状态双向同步：仅对“已关联子任务”的 checkbox 生效，普通 checkbox 不受影响。
4. 子任务标题支持在 `Library` 详情页顶部与 `Sticky` 顶部直接编辑。
5. 重命名后同步更新所有引用位置：
   - 正文中的 `taskLink` 标题
   - 便签书签标题
   - 已关联 markdown checkbox 文本
6. 子任务标题在同一父任务下不支持同名（兄弟子任务唯一，跨父任务允许同名）。
7. 完成态同步遵循“最新用户操作优先”：跨窗口/多入口并发更新时，晚到响应不得覆盖最新勾选结果。
8. 任务树重命名子任务后，父任务正文中的对应链接节点需保持 `taskLink` 样式，不得退化为纯文本。
